## 目标

重构 `data/signals.py`，从"单一RSI+MA+MACD三指标打分"变为"分类差异化决策引擎"。

## 核心设计：两套信号逻辑

### 趋势型资产（assetType = "trend"）

**不做择时**。输出固定建议：

| 条件 | signal | reason |
|---|---|---|
| 持仓为零 + 估值≤60 | 定投建仓 | 目标仓位{x}%，建议分3批建仓，每批间隔1-2周 |
| 持仓偏低（偏离>5pt）+ 估值≤60 | 定投补仓 | 当前低配{x}pt，估值适中，可分批补入 |
| 持仓正常（偏离≤5pt） | 持有 | 买入持有策略，回测α=+{x}pt，不做择时 |
| 持仓超配（偏离>5pt）+ 估值≥70 | 可减仓 | 超配{x}pt且估值偏贵，可小幅调出 |
| 持仓超配 + 估值<70 | 持有 | 虽超配但估值不贵，暂不操作 |

### 震荡型资产（assetType = "oscillation"）

**保留技术择时**，但加上估值确认：

| 技术信号 | 估值分位 | 最终 signal | reason |
|---|---|---|---|
| buy_score≥3 | ≤40（低估/偏低） | **买入** | 技术面共振+估值支撑→强信号 |
| buy_score≥3 | 40-70（适中） | **可加仓** | 技术面看多，估值中性 |
| buy_score≥3 | ≥70（偏贵） | **观望** | 技术面看多但估值不支持 |
| sell_score≥3 | ≥60（偏贵） | **减仓** | 技术面+估值双确认→强减仓 |
| sell_score≥3 | <60 | **观望** | 技术面看空但估值不贵，不急 |
| 其他 | ≤20 | **低估可加** | 技术中性但历史性便宜 |
| 其他 | ≥80 | **高估警惕** | 技术中性但历史性偏贵 |
| 其他 | 20-80 | **持有观察** | 无明确信号 |

### 主动型基金（assetType = "active"）

**读取诊断数据**（如有 `data/active_funds.json`），否则按震荡型处理。

## 实现要求

### 改写文件：`data/signals.py`

1. **读取新增数据源**：
   - `data/valuation.json`（估值分位，P0-①的输出）
   - `data/portfolio.json`（目标仓位 + 当前持仓比例）
   - `data/rebalance.json` 或内部计算偏离度

2. **为每只基金确定 assetType**：
   - 从 `portfolio.json` 的 holdings 中读取 `assetType` 字段
   - 映射：trend / oscillation / active / cash
   - cash（货币基金）跳过，不生成信号

3. **信号输出结构升级**（向后兼容）：
```json
{
  "code": "002963",
  "name": "易方达黄金ETF联接C",
  "asset_type": "trend",
  "rsi14": 39.14,
  "ma5": 2.929,
  "ma20": 3.076,
  "macd_hist": -0.031,
  "macd_trend": "bearish",
  "signal": "持有",
  "signal_type": "trend_hold",
  "reason": "趋势型资产，回测证实买入持有年化25.6%优于择时8.6%，不做择时操作",
  "valuation_verdict": "中性偏贵",
  "valuation_score": 62,
  "deviation_pt": +25.3,
  "action_detail": "当前超配25pt，但黄金估值62分(中性偏贵)，可考虑小幅减至目标24%",
  "buy_score": 1,
  "sell_score": 2
}
```

4. **新增字段说明**：
   - `asset_type`：趋势/震荡/主动/现金
   - `signal_type`：具体的信号子类型（trend_hold / trend_dca / oscillation_buy / oscillation_sell 等）
   - `valuation_verdict`：来自 valuation.json
   - `valuation_score`：0-100
   - `deviation_pt`：偏离目标仓位百分点（正=超配，负=低配）
   - `action_detail`：一句话可操作建议（给推送用）

5. **⚠️ 兜底逻辑**：
   - 如果 `valuation.json` 不存在或读取失败，所有估值相关字段设为 null，走纯技术信号逻辑（向后兼容）
   - 如果 `portfolio.json` 中某基金没有 assetType，默认按 oscillation 处理

### 删除旧逻辑

- 删除原来的 `buy_score >= 3` 统一判定逻辑（已被分类差异化逻辑替代）
- 保留 buy_score/sell_score 的计算（震荡型仍在用），但不再作为趋势型的判定依据
