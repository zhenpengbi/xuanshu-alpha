# GUARDRAILS.md — 玄枢Alpha 建设纪律

## 核心目标

玄枢的唯一目标：辅助毕老师做基金交易决策，带来可量化的理财收益。

## 三问准入检查（每个新需求必须通过）

1. 这个功能会改变操作建议吗？
2. 没有这个功能，会导致错误决策吗？
3. 优先级高于当前P0吗？

任一否定 → 排到P0之后。

- [x] 组合风险仪表盘（解决"风险多大"）

## P0.5 清单（已完成）

- [x] 估值兜底：yfinance 失败 → AKShare → 硬编码校准值（QQQ=35, SPY=24），纳指/标普不再"数据缺失"
- [x] 信号阈值校准：buy_score>=2 即可触发"可加仓"/"低估可加"；sell_score>=2 + 高估触发"谨慎持有"
- [x] 同品类信号合并：趋势型按品类整体偏离统一方向（黄金两只不再矛盾）
- [x] 操作具体化：rebalance.json 新增 operations 数组，含具体基金代码/名称/金额/原因

## 当前最大风险（P1 关注点）

- 纳指100/标普500 待建仓，`fund_code = TBD`，需补充 `data/fund_recommendations.json` 或在 portfolio.json 中添加具体基金
- 兜底 PE 校准值为季度人工更新，需建立季度提醒机制（当前 2026Q2 校准值有效期到 2026Q3）
- active 类（高端制造）诊断依赖 active_funds.json，需定期运行 active_fund_diagnose.py
- 信号全为"谨慎持有/持有"时，buy 指令来自仓位偏离而非技术信号，需向用户标注数据来源

## 禁止事项

- P0未完成时不接UI/美化/展示类需求
- 回测结论必须落地为产品动作
- 不得把"展示更多数据"等同于"决策能力提升"

## Claude Code 监督权

- 任务不符合准入条件时，有权拒绝执行
- 发现信号全部相同（如全"观望"）时，主动提醒
